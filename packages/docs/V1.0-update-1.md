## 项目背景

当前项目是一款基于 Socket 实时通信的多人唬牌（Cheat / I Doubt It）游戏。

现有版本采用：

* 标准 52 张扑克牌
* 2~6 人开局
* 服务端自动轮转目标牌点
* 玩家只能选择出牌
* 玩家不能自主声明牌点
* 其他玩家可发起质疑

为了提升心理博弈深度和玩法自由度，需要对房间系统和核心出牌规则进行改造。

---

# 一、房间人数系统改造

## 当前实现

创建房间时：

* 服务端自动生成房间码
* 房间最大人数固定为 6
* 开局支持 2~6 人
* 房主可随时开始游戏

存在问题：

* 双人局容易被其他玩家加入
* 三人局无法锁定人数
* 无法快速形成固定规模对局

---

## 目标实现

创建房间时允许房主选择房间人数。

支持：

* 2人房
* 3人房
* 4人房

不再支持：

* 5人房
* 6人房

---

## 前端改造

创建房间弹窗新增：

房间人数

○ 2人

○ 3人

○ 4人

默认：

4人

---

创建房间请求新增：

```json
{
  "maxPlayers": 4
}
```

## Room 数据结构

新增：

```ts
interface Room {
  id: string;

  roomCode: string;

  ownerId: string;

  maxPlayers: number;
}
```

允许值：

```ts
2
3

```

## 加入房间校验

新增：

```ts
if (room.players.length >= room.maxPlayers) {
  throw new Error("房间已满");
}
```

## 大厅展示

显示：

```text

2/4

1/2

3/3（已满）
```

## 等待房间展示

显示：

```text

```

## 开始游戏条件

改为：

```ts
players.length === room.maxPlayers
```

例如：

2人房：

```text

```

允许开始。

3人房：

```text
3 / 3
```

允许开始。

开始按钮：

```ts
canStart =
players.length === room.maxPlayers
```

# 二、核心玩法改造

## 当前实现

当前流程：

```text
玩家选择牌
↓
系统自动根据目标牌点声明
↓
进入下一轮
```

例如：

当前目标牌点：

A

玩家打出：

♠A ♣A

系统自动声明：

2张A

## 存在问题

玩家无法主动撒谎。

游戏策略完全围绕：

* 有没有目标牌点
* 是否要赌别人质疑

缺少经典吹牛牌玩法中的：

* 自由声明
* 主动欺骗
* 心理博弈

# 九、最后一手可质疑

## 当前实现

玩家出完最后一张牌：

```text
立即获胜
```

导致：

```text
最后一手无法被质疑
```

---

## 新规则

玩家出完最后一张牌后：

进入待确认状态。

```ts
pendingWin = true
```

## 状态示例

```ts
interface MatchPlayer {
  handCount: number;

  pendingWin: boolean;
}
```

---

## 结算逻辑

### 情况1：无人质疑

玩家维持空手牌状态。

进入胜利结算。

---

### 情况2：被质疑且为真

立即获胜。

---

### 情况3：被质疑且为假

吃回弃牌堆。

```ts
pendingWin = false
```

继续游戏。
